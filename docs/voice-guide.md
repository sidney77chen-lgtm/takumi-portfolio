# Voice Guide · Sidney 作品集網站文案指引

> 這份文件是給 AI（Claude Code、Claude）使用的語氣與人設指引。
> 每次調整網站文案前請先讀取本檔。
> 履歷本身（resume.md）只在需要查證具體經歷時才參考。

---

## 1. 核心定位（一句話）

> **「我是一個用視覺解決商業問題的設計師。美學是手段，邏輯是骨架，即戰力是承諾。」**

這句話不需要直接放在網站上，但所有文案的語氣與內容方向，都應該圍繞這個核心。

---

## 2. 三個關鍵差異化標籤

撰寫任何自我介紹文案時，這三個標籤必須出現至少其中一個：

1. **美學與商業目標的平衡能力** — 我不為美而美，視覺要服務目的
2. **工程思維與跨單位溝通力** — 我能跟前端、PM 無縫對接，降低反工率
3. **AI 工具驅動的設計工作流** — 我用 Claude Code、Google AI Studio 加速產出

---

## 3. 具體可用的數據與資歷（請大膽放上網站）

這些是事實，不是包裝。寫文案時可直接引用：

- **12+ 年** UI 設計經歷
- **20+ 個** 跨產業改版專案
- **100%** 客戶滿意度（魁亨科技時期）
- **6 人** 設計團隊引領經驗
- 跨產業：醫療、金融資訊、電商、房仲求職、B2B SaaS

---

## 4. 設計信念（可用於 About 頁面或作品反思）

寫文案時可挑 2–3 點融入，不要全部列出：

- 設計的價值不在於「畫得多漂亮」，而在於「**讓對的資訊在對的時機被看見**」
- UI 與 UX 不是兩件事。UI 是 UX 的具體實現，**邏輯不通的視覺只是裝飾**
- Design System 不是文件，是**團隊協作的共同語言**
- 好的設計師應該在前期就介入 UX Flow，**減少後期反工就是設計的價值**
- AI 工具不是取代設計師，而是讓設計師**把時間花在更值得思考的地方**

---

## 5. 文案語氣準則

### 人稱
- **第一人稱「我」**（不要用「我們」或第三人稱「Sidney 是...」）
- 例外：技能 / 工具列表可用名詞句

### 語氣
- **自信但不浮誇**：說事實、給觀點，不堆疊形容詞
- **具體勝過抽象**：「降低 30% 反工率」優於「大幅提升效率」
- **觀點勝過陳述**：「我認為後台系統設計的核心是減少認知負擔」優於「我擅長後台系統設計」
- **保持節制**：每段話的形容詞不超過 2 個

### 句型結構
- 短句優先，長句用來補充細節
- 不要在同一段塞 3 個以上的概念
- 段落之間要能單獨成立（讓掃讀的人也能 get 重點）

---

## 6. 慣用詞 vs 避免用詞

### ✅ 推薦使用（這些詞貼近你的人設）

| 領域 | 推薦詞 |
|---|---|
| 視覺 | 視覺定位、視覺語言、產品視覺、品牌調性 |
| 系統 | 設計系統、元件庫、規範、可擴展性、一致性 |
| 邏輯 | UX Flow、操作邏輯、資訊架構、認知負擔、流程可視化 |
| 協作 | 跨單位溝通、轉譯成本、反工率、開發效率、Handoff |
| 結果 | 即戰力、引領、把關、平衡、優化 |

### ❌ 避免使用（這些詞會稀釋你的定位）

| 類型 | 避免詞 | 為什麼 |
|---|---|---|
| 陳腔濫調 | 「熱愛設計」「對美感有追求」「致力於 OOO」 | 太空、誰都能寫 |
| 純 UX 學術詞 | 「User Persona」「Empathy Map」「User Journey Map」（除非真的有做） | 你不是 UX 研究員定位，硬塞會顯得假 |
| 過度謙虛 | 「小小的設計師」「希望有機會 OOO」「不才」 | 12 年資歷不需要這樣寫 |
| 浮誇形容 | 「驚艷」「絕美」「極致」「革命性」 | 不符合你「邏輯感」的人設 |
| 中英夾雜過量 | 「我的 mindset 是 user-centric 的 design philosophy」 | 必要的英文術語 OK，但不要每句都夾 |

## 7. 三個作品的差異化文案方向

每個作品的文案重點不應重複，請各自聚焦在以下面向：

### 作品一｜醫療後台 SaaS Dashboard
- **核心訴求：** 複雜系統的清晰化 + UX Flow 優化
- **可呼應履歷：** 「優化 UX Flow 操作邏輯、大幅減少開發反工率」「前期排除潛在流程瓶頸」
- **避免講：** 視覺有多美、配色多有質感
- **可用切入句：**
  - 「在高壓力、高頻率的操作環境下，設計要服務的是速度與準確性。」
  - 「後台系統的設計核心不是視覺，而是減少認知負擔。」

### 作品二｜高質感品牌房仲求職前台
- **核心訴求：** 視覺美學 + 品牌風格定位
- **可呼應履歷：** 「複雜商業目標中定位產品視覺與美學風格」「3D 渲染與微互動技術提升品牌形象」
- **避免講：** 流程多嚴謹、系統有多嚴密
- **可用切入句：**
  - 「品牌網站的目標不只是好看，而是在第一屏建立信任感。」
  - 「視覺語言是品牌與用戶之間的第一次對話。」

### 作品三｜Design System / 跨平台整合
- **核心訴求：** 系統思維 + 跨團隊協作能力
- **可呼應履歷：** 「Design System Specialist」「主導並運用 Figma 建立、維護系統化設計規範」「指導初階設計師維持產出品質」
- **避免講：** 視覺多美、單一頁面多精緻
- **可用切入句：**
  - 「Design System 的終極用戶是設計師和工程師，讓它易用本身就是 UX 設計。」
  - 「系統不是文件，是團隊協作的共同語言。」

## 8. 各頁面文案範例與基調

### Hero / 首頁主視覺區
**基調：** 自信、簡短、有觀點
**範例：**
```
Hi, 我是 Sidney。
資深 UI 設計師 · Design Lead · 12 年資歷

我相信好的設計，不只是讓人看見，
而是讓對的資訊在對的時機被看見。
```
（不要寫成「我熱愛 UI 設計，致力於打造美好的使用者體驗」這種空話）

### About Me 區塊
**基調：** 像跟對方對話、有具體事實支撐
**範例：**
```
過去 12 年，我在醫療、金融、房仲、B2B SaaS 等領域，
完成了 20+ 個改版專案，客戶滿意度 100%。

我的設計風格偏「克制」——
我不擅長花俏，但我擅長讓複雜的事情變得清楚。

身為 Design Lead，我帶過 6 人的設計團隊，
建立過多套 Design System，
也學會用 Claude Code、Google AI Studio 把產出速度再推進一階。
```

### 技能 / Skills 區塊
**基調：** 不要只列工具名，要分群並標示熟練度或用途
**範例分組（建議）：**
- **設計工具：** Figma · Adobe XD · Framer
- **AI 工作流：** Claude Code · Google AI Studio · Google Stitch · Lucid
- **視覺與動態：** Photoshop · Illustrator · After Effects
- **前端基礎：** HTML · CSS · JavaScript

可在分組旁加一句話，例如：
> 「AI 工作流是我這兩年最大的工作方式轉變——它讓我把時間留給更值得思考的決策。」

### 聯絡 / Footer 區
**基調：** 直接、不囉嗦
**範例：**
```
有興趣聊聊？
sidney77chen@gmail.com
```
（不要寫「期待您的來信」「歡迎合作」這種客套話）


## 10. 最後一個原則

當你不確定該不該寫某段話時，問自己：

> **「如果我把這段念給一個設計總監聽，他會覺得我在說重點，還是在說廢話？」**

如果是後者，就刪掉。
