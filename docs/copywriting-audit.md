## ⚠️ 先講三個跨頁的結構性問題（影響全站，建議優先處理）

### A. 年資自相矛盾，且嚴重低估
- `index.html` Hero（index.html:334）寫「**4+ 年** B2B SaaS 產品設計」；指標區（index.html:749）寫「**4+** Years Experience」。
- 但 `resume.md` 與 voice-guide 第 3 節明寫「**12+ 年** UI 設計經歷」。
- **這是最傷的硬傷**：網站鎖定設計總監／資深職缺，卻把自己寫成資淺者，且與其他頁（about 寫「十年後回到工藝」）互相打架。**優先度：高**

### B. `about.html` 整頁是虛構人設，與真實履歷衝突
- 經歷區（about.html:167-183）列出 **完全虛構** 的「Staff Designer @ Takumi Labs」「Design Director @ Horizon Global（管理橫跨三大洲 15 位設計師）」「Senior UI Engineer @ Artisan Interactive」。
- 敘事區（about.html:125）寫「我曾在一家全球性公司擔任五年的設計總監，將產品從零擴展到數百萬用戶」。
- 真實履歷是：股感資訊（現職）、魁亨科技、康闓科技；6 人團隊；100% 客戶滿意度。**虛構資歷不只違反 voice-guide，更是面試風險。** 此頁需整段重寫，優先度最高。**優先度：高**

### C. 同一個作品在不同頁有 3 種名字與 3 種年份
案例一的稱呼橫跨各頁不一致：
- index 卡片：「醫療後台平台 — 多角色權限系統重設計」（chip 標 **2024**）
- 多數 dropdown／footer：「Medical SaaS 平台重設計 · **2024**」
- `selected_works.html` 頁面標題與 brand_identity_2 dropdown：「兒童成長骨科醫療資訊系統 · **2026**」（meta 週期 2025.10–2026.01）

團隊與時程也對不上：index 寫「3 mo. / 1 PM · 2 RD」，selected_works 寫「PM×1·UI×1·前端×2·後端×2」。**這是 meta 文案一致性問題，會讓導覽顯得不可信。優先度：高**

# 摘要

### 總計需調整處
約 **18 個區塊／項目**，分布在 6 個頁面 + 共用 nav/footer。

### 高優先度（共 7 項，建議最先做）
1. **年資 4+ → 12+**（index Hero + 指標區，問題 A）
2. **about.html 虛構經歷整頁重寫**（問題 B，事實造假，最高風險）
3. **三作品命名／年份／團隊全站統一**（問題 C）
4. **首頁補上 AI 工作流技能**（三大差異化標籤之一完全缺席）
5. **首頁 Hero 加入具體事實／差異化標籤**
6. **nav CTA 用語統一 + footer 作品名統一**
7.（順帶）**brand_identity_1 OG description 英文錯置修正**——可獨立快速處理

### 中優先度（5 項）
- 首頁 Designer Profile 定位修正、職人之道語氣收斂、引言去重
- methodology 全頁「我們→我」+ 去學術文藝詞
- 案例一收斂浮誇形容

### 低優先度（語氣微調）
- 全站 Final CTA 副標收斂、案例三（已達標，僅作為基準範本）

### 建議執行順序
1. **先定「事實層」**：年資、about 真實經歷、三作品統一命名表（先建一張「正確名稱／年份／角色」對照表當 single source of truth）。→ 解決問題 A/B/C，止血最大風險。
2. **再補「內容缺口」**：AI 工作流技能、Hero 差異化標籤。
3. **最後做「語氣層」**：methodology、about 敘事、職人之道等文藝腔收斂，**以 brand_identity_2 的語氣為全站基準**。
4. 共用 nav/footer 一致性收尾（含 CTA 用語）。

> 依 CLAUDE.md「一次只改一個檔案」規範，實作階段建議每頁一個 commit；about.html 因需大改＋查證履歷，單獨成一輪最安全。
